## 記事内で活用すべきZenn固有のマークダウン記法

Zennでは標準Markdownに加え、以下の独自記法が使える。記事の読みやすさと訴求力を高めるため、積極的に活用する。

### メッセージボックス（:::message）

重要な概念や定義の強調に使う。

```markdown
:::message
**AIネイティブ開発の定義：** AIを開発チームの構成員として組み込み、人間とAIの役割を構造的に再設計した開発手法。
:::
```

**推奨する使用箇所：**
- 序章：AIネイティブ開発の定義
- 第1章：オペレーターの定義
- 第3章：Phase 1の完了条件「顧客の真の課題が1文で言語化できていること」
- 第3章：Phase 0〜2 総合ゲートの2条件
- 第7章：最上位原則「ユーザーの意図を完遂させる」

### 警告メッセージボックス（:::message alert）

品質プロセスの省略禁止や、牽制構造の破壊につながる行為への警告に使う。

```markdown
:::message alert
ロールの統合・兼務は相互牽制の前提を破壊するため、能力的に可能であっても行わない。
:::
```

**推奨する使用箇所：**
- 第2章：「能力的に可能であっても統合しない」の強調
- 第5章：「速度が上がっても品質プロセスを省略しない」の強調
- 第6章：「AIのレビュー結果を無条件で受け入れない」の強調
- 第5章：Phase 7 完了条件の「4つすべてがOKでなければMVPリリースを許可しない」

### アコーディオン（:::details）

補足的な説明や具体例を折りたたみで提示する。本文の流れを妨げずに詳細を提供できる。

```markdown
:::details 5 Whysの具体例
- なぜ日報を集計したいのか → プロジェクトごとの工数を知りたい
- なぜ工数を知りたいのか → 利益率を把握したい
- ...
:::
```

**推奨する使用箇所：**
- 第3章：5 Whysの掘り下げ例
- 第4章：API設計の癒着パターンA・B
- 第5章：フィードバック収集の「良い聞き方・悪い聞き方」
- 第5章：フェーズ間の差し戻しルール表

### 引用（blockquote）

完了条件やゲート条件は既に引用記法を使用している。Zennでは引用が視覚的に際立つため、現状の記法を維持する。

## 連載としての投稿戦略

### 記事間リンク

各記事の冒頭と末尾に、前後の記事へのリンクを追加する。

```markdown
---
**← 前の記事：** [序章：AIネイティブ開発とは何か](リンク)
**→ 次の記事：** [第2章：5つのAIロールと相互牽制の設計原則](リンク)
---
```

### 連載目次の掲載

序章の「本書の構成」セクションを、全記事の冒頭に短縮版として掲載する。現在読んでいる章を太字にする。

```markdown
> **AIネイティブ開発 連載目次**
> 序章 / 第1章 / **第2章（この記事）** / 第3章 / 第4章 / 第5章 / 第6章 / 第7章
```

### Zenn Book化の検討

全8記事の連載は、ZennのBook機能（チャプター構成の電子書籍）としてまとめることも検討に値する。Bookにすれば目次付きで一気通貫で読めるため、連載の構造的な性質と相性が良い。
