## プロジェクト概要

本リポジトリは、12年のフルスタックエンジニア経験に基づいて確立された**「AIネイティブ開発手法」**を体系化し、Zennでの記事投稿・収益化を目指すプロジェクトである。

### AIネイティブ開発手法とは

「AIを使って開発する」のではなく、**AIを開発チームの構成員として構造的に組み込む開発手法**。以下の3つが核となる設計思想：

1. **人間（オペレーター）は意思決定者** ── 作業はAIが担い、人間は判断・承認・差し戻しに専念する
2. **5つのAIロールによる相互牽制** ── PM、コーディングエージェント、コードレビュアー、システム監査官、ユーザー・運用サポートを独立して運用し、単一視点では見落とす問題を構造的に捕捉する
3. **9つの開発フェーズとゲート条件** ── 各フェーズに明確な完了条件を設け、特にPhase 0〜2（課題発見）を厳格に管理する

最上位原則は **「ユーザーの意図を完遂させる」**。

## Zenn投稿による収益化計画

### 収益化の方針

Zennでは**通常の記事（Article）を単体で有料販売することはできない**。収益化には以下の選択肢がある：

| 方法 | 内容 | 収益性 |
|------|------|--------|
| **無料記事 + バッジ（投げ銭）** | 8本の記事を個別に無料公開。読者からバッジを受け取る | 低〜中 |
| **有料Book** | 8章構成のBookとして200〜5,000円で販売 | 中〜高 |
| **併用（推奨）** | 序章〜第2章を無料記事で公開し認知を獲得 → 全8章をBookとして有料販売 | 高 |

### 有料Book化の手順

1. Zennにログイン →「投稿する」→「本を作成」
2. タイトル・紹介文を入力
3. 各章をチャプターとして追加（00〜07の8チャプター）
4. 価格を設定（200〜5,000円、100円単位）
5. 一部チャプターを無料公開に設定（序章・第1章を無料にすると集客効果あり）
6. カバー画像を設定して公開

**Zenn CLIでGitHub連携する場合：**

```
books/
  ai-native-dev/
    config.yaml       ← タイトル・価格・チャプター順序
    cover.png          ← カバー画像
    introduction.md    ← 序章
    operator.md        ← 第1章
    ai-roles.md        ← 第2章
    ...
```

`config.yaml` の例：
```yaml
title: "AIネイティブ開発 実践ガイド"
summary: "5つのAIロールと9つの開発フェーズによる、AIを開発チームの構成員として組み込む実践手法"
topics: ["ai", "開発手法", "チーム開発", "コードレビュー", "設計"]
published: true
price: 1000
chapters:
  - introduction
  - operator
  - ai-roles
  - phase0-2
  - phase3-5
  - phase6-8
  - review-structure
  - principles
```

**注意：有料Bookの場合、GitHubリポジトリはプライベートにすること。** パブリックのままだとコンテンツが無料で読めてしまう。

### 各記事のZennメタデータ

`zenn-publishing-guide.md` に記事ごとの以下の情報が定義済み：
- **トピック（topics）** ── 最大5個、記事内容に即して選定済み
- **アイキャッチ絵文字（emoji）** ── 各章のテーマを視覚的に表現
- **タイプ（type）** ── 全記事 `tech`
- **トピック選定理由** ── 各記事にどのような読者層を想定しているかの根拠

### 記事への推奨差し込み（未実施）

`zenn-publishing-guide.md` に定義済みだが、まだ記事本体への反映は行っていない：

- **`:::message`**（メッセージボックス） ── 定義・完了条件・最上位原則の強調（5箇所）
- **`:::message alert`**（警告ボックス） ── ロール統合禁止・品質プロセス省略禁止の警告（4箇所）
- **`:::details`**（アコーディオン） ── 5 Whys例・API癒着パターン等の折りたたみ（4箇所）
- **連載目次** ── 各記事冒頭への短縮目次の追加
- **前後記事リンク** ── 各記事冒頭・末尾へのナビゲーション
